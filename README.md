# GoalReal — BeReal for Goals 🔥

A social accountability platform where small communities keep each other consistent. One goal. One daily proof photo. Real streaks.

**Built for the buildathon · Production-quality MVP · Zero external dependencies to demo**

---

## ⚡ Quick start (2 minutes)

Requirements: **Node.js 18, 20, or 22 (LTS)**

```bash
npm install
npm run build
npm start
```

Open **http://localhost:3000** — that's it.

The SQLite database auto-creates and self-seeds on first launch with 20 users, 5 communities, 900+ posts, likes, comments, streaks, and notifications. The app looks alive from second one.

> For development: `npm run dev`

## ✨ What's inside

- **Landing page** — hero, features, how-it-works, testimonials, FAQ, CTA
- **Auth** — email/password (scrypt-hashed), signed HTTP-only session cookies, middleware-protected routes, instant demo login
- **Daily proof posts** — one per day, enforced at DB level (unique constraint) *and* API level; photo upload or auto-generated category art
- **Social** — likes, comments, follows, milestone celebrations, notifications
- **Streaks** — current, longest, 30-day consistency, GitHub-style heatmap calendar
- **Communities** — create/join/leave, private communities with join requests, banners, member walls, per-community feeds & stats
- **Leaderboards** — weekly, ranked by streak + consistency + volume
- **Analytics** — weekly activity, 8-week consistency trend, posting calendar
- **AI Weekly Coach** — personalized weekly insights ("Friday is your weakest day", "you improved 14%") with one actionable suggestion
- **AI Accountability Partners** — match scores based on goal, consistency & schedule similarity
- **AI Community Insights** — weekly engagement pulse per community
- **AI Challenge Generator** — auto-generated weekly community challenges
- **AI Moderation** — spam/toxicity auto-flagging on posts & comments

### How the AI works

Set `OPENAI_API_KEY` in `.env` (copy `.env.example`) to have coach insights generated by OpenAI (configurable endpoint/model via `OPENAI_BASE_URL` / `OPENAI_MODEL`) and moderation run through OpenAI's moderation endpoint.

**Without a key, nothing breaks**: a built-in deterministic insight engine computes the same reports from real posting data (day-of-week gaps, posting hours, week-over-week deltas). Insights are cached per week in the `ai_insights` table. This means the live demo has zero API-failure risk.

## 🧱 Tech stack

Next.js 14 (App Router) · React 18 · TypeScript · Tailwind CSS · Framer Motion · SQLite (better-sqlite3) for the local demo · OpenAI API (optional) · Supabase schema included for cloud deployment

## 🗃️ Data model

`users` · `communities` · `memberships` · `posts` · `likes` · `comments` · `follows` · `notifications` · `challenges` · `ai_insights` — full relational schema with foreign keys, unique constraints (one post/user/day, one like/user/post) and indexes. See `src/lib/db.ts` (SQLite) and `supabase/schema.sql` (Postgres).

## 🚀 Deploying to Vercel + Supabase

The demo runs on SQLite so it works anywhere with zero setup. For cloud deployment, see **[DEPLOY.md](./DEPLOY.md)** — it includes the Postgres schema (`supabase/schema.sql`) and a step-by-step guide (~15 minutes).

## 📁 Project structure

```
src/
  app/            # App Router pages (landing, login, feed, communities, profile, analytics, coach, notifications)
  components/     # PostCard, Composer, Heatmap, Charts, AppNav, ...
  lib/
    db.ts         # SQLite connection + schema + auto-seed
    seed.ts       # Deterministic demo data generator
    auth.ts       # Sessions + password hashing
    actions.ts    # All server actions (posts, likes, comments, communities, follows)
    ai.ts         # Weekly coach, partner matching, community insights, moderation, challenges
    stats.ts      # Streaks, heatmap, leaderboard, trends
    queries.ts    # Read queries
```
